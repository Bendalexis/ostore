StoreTable

trait StoreTable extends AnyRef

A local store table contains OStore items and control information. Each local store table has three key-value parts

  • meta. Per item metadata. Keys here are ordered.
  • vals. Per item values. Keys here need not be ordered,
  • control. System control information. Keys here need not be ordered.

Keys and values are each strings.

For each OStore item, its key is used to look up its metadata in meta and its value in vals. If the item is a tombstone there will be a meta value but no vals value.
